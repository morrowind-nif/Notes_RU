Animation Kit

Программка для быстрого слияния КФ файлов, без участия 3д МАХа.
Позволяет подклеивать новые файлы анимаций в БайзАним.
Позволяет разбивать КФ файлы на отдельные анимаций.
Позволяет подклеивать эти самые отдельные файлы в базовый файл анимации.
Что, в теории, можно использовать для добавления анимации существам, добавляя эти Кфки к их основной анимации.
Увы, работает не всегда и не везде.
 
Примечание.
Извлеченные из Базовой анимации КФ файлы, можно использовать в 3д МАХе.
Сиречь, добавляя оные в желаемой последовательности, либо редактируя оные а затем подшивая в базовый файл анимации.
См. тут.
В целом, обработка в МАХе позволяет куда более надежно собрать свой собственный файл анимации.
Хотя, это займет много больше времени, чем пользование Animation Kit (встроенная утилита merge_kf).
 
Примечание по Animation Kit конкретно.
- не понимает XYZ rotation keys, т.е. если в файле существа будут такие ключи, разбиения не произойдет.
- утилиты работающие с КФ файлами подвязаны на путь в реестре к каталогу игры, если путь не указан - отказываются работать.
- подвязка на реестр, существенно осложняет работу с программой. Особенно если установлено несколько копий МВ.
- графическая оболочка позволяет только приклеивать новые файлы, но не позволяет сливать, или разделять другие КФ файлы.
- требует 7ку. Под ХРями отказывается работать, видимо из принципа.
- работа по разделению и слиянию КФ файлов, консольная.
- после каждого действия предлагают Выйти, что не совсем удобно если требуется продолжить работу.
- вообще не работает с файлами содержащими XYZ rotation keys. Ни просмотра, ни слияния, ни разделения.
К сожалению, утилита явно не доделана.
Но, исправления данной ситуации, похоже, не предвидится. Автор давно отошел от дел.
 
В комплекте.
- Графический интерфейс для слияния с Базовой анимации других "кошерных" файлов.
- консольные утилиты:
split_kf.ехе - для разделения одного КФ файла на множество, согласно группам анимаций в нем.
preview_kf.ехе - для просмотра анимации.
merge_kf.ехе - для слияния.
 
Как с этим работать:
- что бы получить "кошерный файл" (который затем примет ГУЙ АнимКита), требует обработать целевой КФ посредством split_kf.ехе, в итоге должна получится папка, в которой будут лежать отдельные файлы анимаций.
Эту папку надо положить в \Data Files\Animation\.
По идее, после этого действия, в Лаунчере АнимКита, будет доступен новый тип анимации, которую можно будет подклеить к Базовому Файлу анимации.

DDS Save Options

примечания по значениям для 5.12 модуля ДДС.
*в целом практически одинаково и для всех старших версий модуля.
 
Save format
Выбор формата текстуры.
Т.е. качества, или включение особых настроек файла.
Позволяет выбрать качество альфа канала, отключить оный, или настроить иные специфические моменты.
 
DXT1 (no alpha)
сжатый формат сохранения текстур БЕЗ альфы (сжатие до 8 раз).
 
DXT1 (1 bit alpha)
сжатый формат сохранения текстур С 1-битной альфой (сжатие до 8 раз).
 
DXT3 (Explicit Alpha)
сжатый формат сохранения текстур С альфой, имеющей индексированную палитру меньше 256 цветов (сжатие до 4 раз).
 
DXT5 (Interpolated Alpha)
сжатый формат сохранения текстур С полноцветной альфой (сжатие до 4 раз). Т.е. градиент альфы максимально ровный.
 
16 bit (4.4.4)
несжатый формат 4:4:4:4
 
16 bit (1:5.5.5)
несжатый формат 1:5:5:5
 
32 ARGB
несжатый вариант сохранения текстур С альфой (максимальное качество текстуры + размер)
 
Palette 8-16bit
несжатый палитровый формат 8 бит (требует включения INDEX COLOR в настройках режима отображения)
Иначе, при попытке сохранить такой файл, ДДС плагин зависнет.
Т.е. сначала переключиться на Indexed Color и только затем сохранять текстуру.
 
Mip Maps
Создание мип-уровней.
Можно создать с нуля, можно оставить те, что уже были созданы, или отказаться от их создания.
 
См. раздел по работе с оными.
 
Generate Mip maps
автоматически создает мип уровни при сохранении файла
 
Specify # MIP maps
указывает число мип уровней, т.е. можно задать только определенное кол-во.
 
Use existing Mip Maps
если мип-уровни были созданы, они не будут изменены при пересохранении файла.
 
No Mip Maps
мип-уровни не создаются. Это удобно при создании текстур для текстурых эффектов, которые не используют мип-уровни.
 
Mip Map generation
Отвечает за настройки фильтрации мип-уровней.
 
 
См. здесь.
 
Option
Позволяет поиграть с некоторыми тонкими настройками.
Например создать рамку, или добавить дизеринг.
 
Dither color
Включает дизеринг цвета, фактически влияет на альфа канал.
Формат сохранения играет роль! Если ARGB32 размытие альфа сохраняется максимальным.
Для DXT1-5 дизеринг, будет играть роль!
выкл (стандартное значение) (DXT5)
вкл
ARGB32, флаг дизеринга установлен, т.к. включен.
DXT3 вкл.
 
Dither each MIP
включает дизеринг по мип-уровням.
Фактически аналогично выше показанному, только работает по мип-уровням.
Флаги взаимосключающие! Или цвет, или уровни.
 
Binary Alpha
По видимости убивает весь градиент альфы.
Формат роли не играет (т.е. ARGB32 или DXT5)
оригинальный альфа канал.
и после сохранения с флагом Binary.
 
Grey Scale
Накладывает фильтр на текстуру, в старой версии (5.12) не делает ничего заметного.
 
Alpha Zero border 1 texel
Создаст рамку по контуру изображения толщиной в 1 пиксель.
По идее должна влияет на альфа канал.
 
Border Color 1 texel
Border Color
Создаст рамку означенного цвета по контуру изображения.
Определяет цвет рамки.
(сомнительные опции в большинстве случаев)
т.е. по контуру изображения пойдет рамка в 1 пиксель белого цвета, если оный цвет не был указан особо.
 
 
Normalize MIP Maps
Должно влиять на смешивание мип-уровней.
Normalize mip maps - по идее, можно ставить в целях оптимизаций.
 
DuDv Map
Фактически является фильтром.
Т.е. автоматически создает пресет для карты искажения шейдера.
Заточено под создания шейдера воды, где, подобная карта искажает свет.
Также Nvidia DDS плагин может создавать такую карту на стадии генерации карты нормалей.
 
 
и вот так это влияет на текстурный эффект.
 
Fade MIP Maps
Затенение мип-уровней.
Можно задать изменение цветности текстуры, или смешивания альфа в зависимости удаления от камеры.
Blend Amount - смешивание мип-уровней, большие значения дают забавные эффекты.
 
 
Фактически является эффектом, которым можно дополнительно разнообразить поведение модели.
Меняет цвет и прозрачность модели автоматически.
Т.е. установив  пороги смешивания в желаемом значении, можно несколько сократить время получения эффекта.
См. этот раздел.
 
 
Select Format on Preview
Позволяет настроить предпросмотр текстуры в окне Фотошопа (Гимпа).
Не влияет на сохранение.
 
Preview Options
Опции предпросмотра текстуры.
Не влияет на сохранения.
 
Types of texture.
Позволяет указать тип сохраняемой текстуры.
2д - обычное значение, должно быть всегда кратно 2х.
Cube Map - создает кубическую карту, требует особых настроек.
Image - текстура может не быть кратна 2х.
 
Config
Ask to Load Mip maps - позволяет загрузить файл с мип-уровнями.
Т.е. они будут видны при загрузке файла в Фотошоп.
Обычно эту опцию отключают при первом старте, дабы не мешалась.
Но для работы с лодами текстуры, ее придется ставить обратно.
 
Show Size warning - если не стандартный размер текстуры, будет дано предупреждение, иначе - просто блокировка сохранения.
 
Show alpha warning - если текстуру с альфа каналом сохраняют в формате ДХТ1 будет выдано предупреждение.
Обе опции лучше держать включенными!
 
Примечание.
Делается это в меню настроек сохранения файла.
Т.е. надо вызвать сохранение, поменять флаг, закрыть файл (можно не сохраняя) и снова загрузить файл в ФШ.
Теперь он будет загружаться с включенными лодами.
Но наиболее удобно, создать две копии фотошопа!
В одной сделать настройки ДДС с отображением ЛОДов.
В другой отключить это дело.
Теперь вы сможете выбирать нужный режим работы запуском другой копии ФШ.
 
Это совсем махрово старая, но однако, наиболее удобная и стабильная версия. (5.12)
 
Возможно будет удобно использовать эти модули в связке.
И более "новая" (8.2).
Умеет сохранять профили, создает больше форматов ДДС и все опции заныканы под кнопки, что делает ее не сильно удобной в плане эргономики. Также, принудительно создает альфа канал, во всех файлах.
*чистый фон белого цвета. Иногда начиная подглючивать и запихивает туда артефакты в виде черных пикселей.
Впрочем лучше работает с файлами большого размера 4096 и корректно открывает файлы с отображением мип-уровней (лодов), что и позволяет их редактировать.
 
 
Опции управления отображением мип уровней при загрузке и пр. вынесено под кнопку READ CONFIG.
Когда они понадобятся здесь их можно включить.
 
Диалоги сохранения, логично, вызываются только при сохранении файла, вызов без наличия загруженного файла, похоже не предусмотрен.

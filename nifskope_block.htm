Nifskope Block

Block — важное и полезное меню!
insert
вызывает меню «вставки» в котором можно выбрать множество различных опций доступных по формату файлов.
Вставленное появляется за приделами корневой ноды, откуда его придется дополнительно копировать и вставлять.
 
Примечание.
По умолчанию, для МВ использовать можно лишь небольшую часть объектов!
Т. к. значительная часть относиться к старшим Тесам или вовсе Фол-ауту.
Объекты сгруппированы в алфавитном порядке. Префикс ni + первая буква названия объекта.
В целом довольно удобно, если знать, как называется объект для вставки, не составляет труда его найти. Во вложенных меню, объекты также упорядочены по имени.
Исключения: RootNode AvoidNode находятся в разделе Bethesda.
 
Список объектов работающих для Морровинда см. здесь.
Примечание по оптимизации списка объектов, см. здесь.
 
remove
удаляет выделенные свойства из файла вообще!
Т.е.  изо всех мест, где удаляемый объект был использован. 
Обычно это касается свойств шейпов; ниМатериал, ниТекстур, ниАльфа.
В ряде случаев, использование, пополняет кол-во мусорных свойств!
Использовать с осторожностью!
 
Примечание.
ВАЖНО!
Если на объекте доступно только это и нет опции remove branch, значит объект используется в других группах!
Т.е. не уникален!
Если выбрать remove - это удалит ВСЕ записи об этих свойствах из всех прочих групп!
Обращайте на это внимание!
Что бы сделать свойства уникальными, используйте  опцию Split.
Однако, Split - не создает новые записи о текстурах! 
Т.е. номера и записи о ИДе текстур, останутся общими.
И при использовании Remove на текстуре, это удалит ее из всех текстурных свойств, что может привести к багам!
 
copy
копирует только выбранное.
На практике, особой разницы, между Copy Branch может не оказаться.
 
copy branch
самая ходовая опция! Копирует выбранную связь, со всеми ее потрохами и настройками!
Это наипервейший инструмент при обработке Нифов!
 
remove branch
удаляет выделенный объект, со всем содержимым.
 
*в отличие от Remove, которое удалит только сам объект (из всех мест, где он использовался), а его содержимое окажется за приделами корня файла, в так называемых в мусорных свойствах.
*например если удалить Текстурные свойства объекта, то записи о текстурах окажутся именно там.
 
Flatten Branch
вытряхивает все содержимое из ноды.
Кроме прописанных на нее свойств.
Т.е. делает ее пустой, очищая от всех «детей».
 
Доступно только на НиНодах. Шейпы оказываются в детях «материнской» ноды.
Текстурный эффект, удаляется из "детей", но остается на ноде и не сбрасывается!
Т.к. прописан в другой строке: Num Effects.
 
Вытряхнутые шейпы и прочее окажутся в корневой ноде файла устроив там основательный бардак.
 
Move Up
перемещает по списку вверх. В смысле, меняет номер объекта на меньший.
 
Может быть полезным при работе с объектами чувствительных к порядковому номеру в иерархии файла.
Свойства трафарета, эмиттинг частиц по области.
 
Move down
увеличивает номер объекта.
 
Crop branch
обрезает до выделенного объекта.
это бывает удобно для быстрой очистки нифа от всего лишнего.
 
Проще говоря, удалит все остальное!
В файле останется ТОЛЬКО выделенный объект (шейп, нода и пр.)
Будьте очень осторожны с этой опцией!
 
Convert
изменяет тип объекта в определенных приделах.
Если выбрано на Ноде, дает опцию изменения обычной ноды в NiBSAnimationNode, NiBillboardNode и т.п.
Обратное тоже верно.
Т.е. это быстрый способ сменить тип объекта без копипасты и добавления новых нод.
Тоже самое и для Шейпов, можно менять их тип.
 
В целом, при правильном использовании, это полезная опция!
 
Примечание.
Однако, далеко не все работает в Морровинде!
Сверяйтесь со списком, прежде чем конвертировать niNode в niRoom например.
Т.е. нельзя делать такого - иначе, ниф файл станет нерабочим!
Возможно это какая-то недоработка нифскопа.
Т.е. некоторые опции не должны появляться для версии ниф файла 4.0
Можно исправить заменой Nif.xml файла на содержащий только доступные для МВ опции.
 
Примечание.
В ряде случаев, неудачная конвертирование может убить Ниф файл!
Игра откажется принимать такие файлы.
А нифскоп перестанет его открывать.
 
Не смотря на возможность добавления\конверсии - нифскоп может спокойно НЕ ПОНЯТЬ, что он только что сделал.
Т.е. убить ниф при сохранении, либо отказаться работать с тем, что он только, что изменил.
К примеру, вот код подобной ошибки.
shape block 1 has multiple data blocks
QFSFileEngine::open: No file name specified
link points to wrong block type
Duplicate
дублирует выделенное.
 
Аналог КТРЛ+Д  в ТЕС КС.
 
Duplicate branch
тоже самое, но более правильно и копируются все связи.
 
Sort by Name
сортировка по имени.
Кажется не работает из этого меню.
 
В версии нифскопа 1.2 вроде бы поправлено.

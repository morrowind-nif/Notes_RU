Import\Export.
Нифскоп способен получить 3ds файлы.
Это некий унифицированый формата меж «максовского» общения.
А также obj файлы, которые используют другие программы и МАХ в т.ч.
 
На деле, это меню быстрой конвертации мешей из одного формата в другой.
 
Конечно, лучше работать на прямую с нифами, чем с их «исходниками».
Но бывают ситуации когда это может оказаться крайне полезным!
Например, чтобы получить мечи из Обломы в Моровинде, без участия 3д редактора.
Т.е. это меню позволяет быстро конвертировать модели из одного формата в другой.
См. статью.
 
Собственно меню импорта.
 
Открываем некую модель из Морки (например меч) и импортируем в нее Обж файл!
 
Экспорт в obj.
 
DAE это разновидность obj.
 
Открываем модель из Обломы (например меч) и экспортируем его в Обж файл!
Который, затем можно импортировать в модель Морки.
 
 
Примечание:
после 1.0 (qt 4.2.3) не работает экспорт/импорт всей модели (только частями, по NiTriShape).
т.е. если вы захотите экспортировать .nif в .obj следует использовать совсем старую версию нифскопа.
 
Подтверждено.
Для самой ходовой версии 1.1.3.
Нормально экспортируется только один шейп из сцены.
Т.е. следует либо объединять шейпы в один, если это возможно, либо собирать модель по частям.
 
Примечание.
Что происходит в версии нифскопа 2.0 не проверялось.
 
New Window
А это важная опция необходимая при работе с несколькими ниф файлами одновременно.
Открытие нового окна через уже запущенную копию нифскопа позволяет копировать и вставлять ноды между нифами! Всегда, запускайте новую копию нифскопа, только через этот пункт!
Если запустить нифскоп через ярлык, или напрямую, опции прямой копипасты между открытыми файлами не будет. Т.е. скопированное не будет вставляться.
Верно для всей линейки 1.х в 2.0 не проверялось.
 
Reload XML
Перезагрузка\обновление параметров текущего ниф файла согласно внесенным в nif.xml правкам.
 
Очень полезная опция на самом деле!
Используется для проверки внесенных на ходу изменений в этот файл!
Т.е. не требуется перезапускать нифскоп, но просто перезагрузить записи и продолжить работу.
Что очень удобно для отладки и поиска различных параметров моделей.
Собственно все достижения юбилейного 2020 года были свершены благодаря этой возможности!
Для теста различных настроек и новых опций, просто не заменимая вещь!
 
Reload XML + NIF
Перезапускает окно нифскопа и перезагружает модель.
Фактически эта опция более полезна и эффективна, чем перезагрузка XML файла только.
 
Обратите внимание, если были сделаны какие-то изменения они не будут сохранены!
Т.е. нифскоп, заново прочитает исходный ниф файл!
Если изменения важны, перед перезагрузкой файла, сохраните его копию!
Но если изменения не важны, или вовсе не требовались, перезагрузка - сбросит все к исходному виду.
 
Также, весьма эффективно для отладки параметров в nif.xml файле и того, что они произвели в Нифе.
If you have made changes to nif.xml, or simply need to reload your model quickly.
Effectively refreshes both at the same time. Shortcut key is Alt-X.
 
XML checker
Сканер ниф файлов на предмет подсказок и прочего.
Просматривает все варианты ниф файлов.
Работает не слишком быстро, но многое зависит от числа потоков и возможностей ПК.
 
Есть консольный аналог RG, но если нет желания пользовать консоль, можно искать через это меню.
RG - ищет любые записи и в каких угодно файлах, при этом, довольно шустро.
DIR - указывает папку для поиска.
Recursive - поиск по подкаталогам и вообще всем файлам в них.
Block Match - показывает меню со списком всех известных Нифскопу, элементов ниф файлов.
Из списка можно выбрать целевой объект для поиска. Либо забить название руками.
Но... будут найдены только объекты известные Нифскопу.
Т.е. если вставить, например NiRendererSpecificProperty и не сделать об этом запись в Nif.xml файле, результат поиска будет нулевым.
Threads - число потоков для поиска, больше лучше.
Run - запускает поиск.
Reload XML - перезагрузит ХМЛ файл. Появятся новые опции для поиска.
Актуально если нужно поискать что-то новое и данные об этом были добавлены в ниф.хмл файл.
 
Примечание.
Отмечены частые зависания поиска в случаях, если в моделях встречаются "непонятные" нифскопу данные.
Ниф файлы повреждены, содержат неверные записи и пр.
Также зависания могут произойти при сканирования большого числа Ниф файлов.
Приходится "убивать" нифскоп в "процессах" диспетчера задач.
Т.е.RG дает более стабильный поиск. Но поиск объекта нифскопом, позволяет сразу открыть нужный файл.
Имеет смысл, перед поиском сохранить текущую модель.
Либо запускать поиск на "пустом" нифскопе.
Т.е. без загрузки ниф файлов.
 
Resourse Files
Путь к BSA файлам игры, либо плагинов.
Имеются в виду только текстуры!
 
Бывает очень удобно указать в этом меню бса файл с текстурами некоего плагина, дабы не извлекать оные.
Если нет распакованных БСА файлов МВ — рекомендуется здесь указать к ним путь.
Automatic Selection - использует путь из реестра к "официально" установленной версии игры.
Мало полезен в случае большого числа сборок и тестовых копий игры.
